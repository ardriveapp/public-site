Stop looking for that needle in a haystack, and use manifests to organize, deploy, and find your files on Arweave.

With Arweave you get [permanent data](https://ardrive.io/can-data-really-be-stored-forever/), but there are limited options to organize your data.  Now, with the [Arweave File System (ArFS)](https://ardrive.io/arfs) and Arweave manifests you can map all of your transactions on Arweave with friendly names – and host your NFT collection, react app, or static website. Arweave manifests enable you to:

Map Arweave Transaction IDs to friendly names (“puppies.jpg” instead of “43t9bn023-9rtgsodfae-nw5505bnbnnghh8\_9330gn55.jpg”)

*   Use friendlier URLs
    
*   Create logical groupings of content
    
*   Reference individual transactions by their file name and/or path
    
*   Easily host static websites, applications, and NFT projects

#### Make Your Manifest

![](/articles/manifests/img-9b88dd5b.jpg)

To use manifests, first upload your assets (e.g. static website content) to a new folder in the ArDrive web app.

Then click the **+NEW** button and select “Advanced” then “New manifest” to add a manifest to the content in that folder.

Manifests open up a whole new world of possibilities for interacting with the permaweb.  And you don’t have to wait to start using them. 

They are now available in the [ArDrive web app](https://app.ardrive.io/) and the [ArDrive CLI](https://github.com/ardriveapp/ardrive-cli).

For technical help with your manifests, please [see our knowledge base](https://ardrive.zendesk.com/hc/en-us/articles/5300353421467-Arweave-Manifests).
