# Getting the price of an upload

File size is used to determine the base fee to upload data to Arweave. The price of the base fee is set by the Arweave protocol code. ArDrive values Credits at 1:1 with AR when uploading. Prices for uploading in Credits are cheaper than uploading in AR because, while a Credit and AR token have the same value, a community tip is added on to all uploads paid for in AR

Users can purchase credits through ArDrive, and then use them whenever they need to upload their files.

**Note:** All file sizes are represented using binary units of measurement (i.e. 1 mB = 1024 kB).

The price of an upload using Credits takes into account multiple factors like the file size, Arweave network data cost, and other fees required by the service.

# Is the price of data consistent?

Not exactly. The Arweave network protocol is always adjusting the data price to maintain sustainable, perpetual storage. The data price is lowered as the network increases its overall capacity.

This can result in a fluctuating data price. However, for most files these fluctuations would only increase or decrease costs by fractions of a cent.
